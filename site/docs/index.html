Chương 01

Giới thiệu #

htn-tunnel là gì?

htn-tunnel là công cụ mã nguồn mở giúp bạn expose một port trên máy tính local ra internet thông qua một URL HTTPS công khai — không cần cấu hình router, không cần IP tĩnh, không cần mở port firewall.

Bạn chạy một lệnh, htn-tunnel tạo ra một đường hầm bảo mật từ máy bạn đến server trung gian, rồi cấp cho bạn một URL như https://abc12xyz.33.id.vn. Ai truy cập URL đó đều được forward về ứng dụng local của bạn.

ℹ
Mã nguồn mở, MIT License

htn-tunnel hoàn toàn miễn phí. Server công cộng tại 33.id.vn dùng được ngay mà không cần đăng ký. Bạn cũng có thể tự host server của riêng mình.

Khi nào nên dùng tunnel?

  • Dev local + chia sẻ nhanh — Bạn đang code một web app và muốn cho bạn bè/client xem thử mà không cần deploy.
  • Webhook testing — Stripe, GitHub, Discord gửi webhook về localhost không được; tunnel tạo URL thật để nhận.
  • Demo cho khách hàng — Chạy một buổi demo trực tiếp mà không cần staging server.
  • Truy cập database từ xa — TCP tunnel giúp connect đến PostgreSQL/MySQL trên máy bạn từ bất kỳ đâu.
  • Test mobile app — App mobile cần gọi API local; tunnel cho địa chỉ HTTPS cố định để test trên thiết bị thật.
  • Share Jupyter Notebook — Chạy notebook local, share link với đồng nghiệp.

Kiến trúc tổng quan

htn-tunnel hoạt động theo mô hình reverse proxy với WebSocket multiplexing. Client (máy bạn) duy trì một kết nối WebSocket đến server. Khi có request từ internet, server forward qua kết nối này về ứng dụng local. 

  Người dùng          htn-server              Máy của bạn
  (internet)          (VPS / 33.id.vn)        (localhost)

  ┌──────────┐         ┌─────────────────┐       ┌──────────────────┐
  │ browser  │         │ nginx (HTTPS)   │       │ app:3000         │
  │ curl     │──HTTPS─▶│ tunnel router  │──WS──▶│ htn-tunnel http  │
  │ mobile   │         │ yamux mux       │       │ 3000             │
  └──────────┘         └─────────────────┘       └──────────────────┘

  https://myapp.33.id.vn  ────────────────────────────▶  localhost:3000

Giao thức mux là yamux — một multiplexer nhẹ cho phép nhiều stream (request) chạy đồng thời trên một kết nối WebSocket duy nhất, giảm overhead và độ trễ đáng kể so với mở kết nối mới cho từng request.

Bắt đầu trong 3 phút #

Bước 1 — Cài đặt

Yêu cầu Node.js 16+. Hỗ trợ Windows, macOS, Linux.

# Cài đặt toàn cục qua npm
npm install -g htn-tunnel

# Hoặc dùng Go (nếu đã có Go 1.21+)
go install github.com/nhh0718/htn-tunnel/cmd/htn-tunnel@latest

# Hoặc tải binary từ GitHub Releases (không cần Node/Go)
# https://github.com/nhh0718/htn-tunnel/releases

Sau khi cài, kiểm tra bằng:

htn-tunnel --version

Bước 2 — Kết nối server (chỉ làm 1 lần)

Chạy lệnh setup để chỉ định server tunnel bạn muốn dùng. Server công cộng miễn phí là 33.id.vn:4443. 

htn-tunnel setup
# Prompt sẽ hỏi: Enter server address:
# Nhập: 33.id.vn:4443

Địa chỉ server được lưu vào ~/.htn-tunnel/config.yaml. Lần sau không cần setup lại trừ khi bạn muốn đổi server.

Bước 3 — Mở tunnel đầu tiên

Giả sử ứng dụng local của bạn đang chạy ở port 3000. Mở tunnel HTTP bằng: 

htn-tunnel http 3000
# Output:
# ✓ Tunnel online
# → https://abc12xyz.33.id.vn
# Forwarding to: localhost:3000
# Press Ctrl+C to stop

Mở URL đó trong browser — bạn sẽ thấy ứng dụng local của mình hiện ra.

✓
Subdomain random hết hạn sau 2 giờ

Anonymous tunnel (không đăng nhập) tự động hết hạn sau 2 giờ và subdomain thay đổi mỗi lần khởi động lại. Để có subdomain cố định, xem Chương 04 và 05.

Để dừng tunnel, nhấn Ctrl+C trong terminal.

Chương 03

Các loại tunnel #

HTTP Tunnel

HTTP tunnel là loại phổ biến nhất — dành cho web app, API server, bất kỳ dịch vụ nào dùng giao thức HTTP/HTTPS. Server tự động cấp một subdomain HTTPS. 

# Expose port 3000 (Next.js, React, Express...)
htn-tunnel http 3000

# Port 8080 với subdomain cố định "api" (cần đăng nhập)
htn-tunnel http 8080:api
# → https://api.33.id.vn

# Port 5173 (Vite dev server)
htn-tunnel http 5173:vite-dev

TCP Tunnel

TCP tunnel forward raw TCP — dùng cho database, SSH, Redis, hoặc bất kỳ giao thức không phải HTTP. Server cấp một port ngẫu nhiên trên domain. 

# PostgreSQL (cần đăng nhập)
htn-tunnel tcp 5432
# → tcp://33.id.vn:51234

# MySQL
htn-tunnel tcp 3306
# → tcp://33.id.vn:51235

# SSH
htn-tunnel tcp 22
# → tcp://33.id.vn:51236

# Connect đến PostgreSQL từ máy khác:
psql -h 33.id.vn -p 51234 -U postgres mydb
⚠
TCP tunnel yêu cầu đăng nhập

TCP tunnel chỉ khả dụng sau khi htn-tunnel login. Anonymous user chỉ dùng được HTTP tunnel.

WebSocket

WebSocket được hỗ trợ tự động trong HTTP tunnel — không cần cấu hình thêm. Server rewrite Origin header để tránh lỗi CORS, và upgrade connection từ HTTP sang WebSocket trong suốt. 

# Vite HMR (WebSocket) hoạt động tự động
htn-tunnel http 5173

# Socket.IO app
htn-tunnel http 3000

# Jupyter Notebook (cũng dùng WebSocket)
htn-tunnel http 8888
ℹ
Yêu cầu version >= 0.4.6 cho WebSocket

Nếu WebSocket không hoạt động, chạy npm update -g htn-tunnel để cập nhật lên version mới nhất.

Subdomain #

Random subdomain (mặc định)

Khi bạn chạy htn-tunnel http 3000 mà không chỉ định tên, server tự sinh ra một subdomain ngẫu nhiên dạng abc12xyz.33.id.vn. Subdomain này:

  • Hết hạn sau 2 giờ không hoạt động
  • Thay đổi mỗi lần tunnel khởi động lại
  • Không yêu cầu đăng nhập
  • Giới hạn 1 tunnel đồng thời / IP (anonymous)

Custom subdomain

Chỉ định tên sau dấu : khi mở tunnel. Yêu cầu đã login. 

# Syntax: htn-tunnel http <port>:<subdomain>
htn-tunnel http 3000:myapp
# → https://myapp.33.id.vn

htn-tunnel http 8080:api-dev
# → https://api-dev.33.id.vn

# Tên subdomain: chỉ chứa a-z, 0-9, dấu gạch ngang (-)
# Độ dài 3–32 ký tự

Giới hạn và subdomain cố định vĩnh viễn

Với tài khoản đăng ký, subdomain custom của bạn được giữ vĩnh viễn — không ai khác claim được subdomain đó. URL luôn giống nhau mỗi khi bạn tunnel.

Đăng ký tài khoản #

Đăng nhập qua browser

htn-tunnel dùng luồng OAuth-style: lệnh login mở browser, bạn đăng ký/đăng nhập trên web, key tự động được ghi về CLI — không cần copy paste thủ công.

1

Chạy lệnh login

Terminal sẽ in ra URL và tự mở browser của bạn.

htn-tunnel login
# Opening browser: https://33.id.vn/auth/login?callback=...
# Waiting for authentication...
2

Đăng ký / đăng nhập trên web

Điền email và password. Nếu chưa có tài khoản, click "Đăng ký" ngay trên trang đó.

3

Key tự động lưu về CLI

Sau khi xác thực thành công, terminal hiển thị thông báo và key được lưu.

# ✓ Logged in as: your@email.com
# Key saved to: ~/.htn-tunnel/config.yaml
⚠
Browser không tự mở?

Copy URL in ra trong terminal và dán vào browser thủ công. URL có dạng https://33.id.vn/auth/login?callback=...

Config file

Tất cả cấu hình được lưu ở ~/.htn-tunnel/config.yaml. Bạn có thể xem trạng thái tài khoản và tunnel đang chạy: 

htn-tunnel status
# Output:
# Server: 33.id.vn:4443
# User: your@email.com
# API Key: htk_••••••••••••••••
# Active tunnels: 2

API key có định dạng htk_... — giữ bí mật, không commit vào git.

Đăng xuất

Để xóa key khỏi máy (ví dụ khi dùng máy chung hoặc đổi tài khoản):

htn-tunnel logout
# ✓ Logged out. Key removed from config.

Use cases thực tế #

Test webhook từ Stripe / GitHub / Discord

Các dịch vụ như Stripe cần một URL HTTPS thật để gửi webhook event. Bạn không thể dùng localhost. Giải pháp: 

# 1. Mở tunnel cho app đang chạy ở port 3000
htn-tunnel http 3000:my-stripe-test
# → https://my-stripe-test.33.id.vn

# 2. Điền webhook URL vào Stripe Dashboard:
#    https://my-stripe-test.33.id.vn/webhook/stripe

# 3. Trigger test event từ Stripe → bạn nhận được ngay ở local

# Tương tự với GitHub webhook:
#    https://my-stripe-test.33.id.vn/webhook/github

Demo app cho khách hàng

Chạy app local, share link cho khách — không cần deploy lên server, không tốn tiền hosting staging. 

# Đăng nhập trước để có subdomain cố định
htn-tunnel login

# Start app
npm run dev   # → localhost:3000

# Mở tunnel subdomain cố định
htn-tunnel http 3000:my-demo
# → https://my-demo.33.id.vn

# Gửi link cho khách: https://my-demo.33.id.vn
# Subdomain luôn là "my-demo" dù restart bao nhiêu lần

Truy cập database từ xa (TCP tunnel)

Muốn connect từ máy khác hoặc tool như DBeaver, TablePlus vào database trên máy dev của bạn: 

# Tunnel PostgreSQL
htn-tunnel tcp 5432
# → tcp://33.id.vn:51234

# Connect từ máy khác bằng psql:
psql -h 33.id.vn -p 51234 -U postgres mydb

# Hoặc dùng connection string:
postgresql://postgres:password@33.id.vn:51234/mydb

# MySQL tương tự:
htn-tunnel tcp 3306
mysql -h 33.id.vn -P 51235 -u root mydb
!
Bảo mật database tunnel

Database TCP tunnel expose trực tiếp port database ra internet. Luôn dùng mật khẩu mạnh và chỉ mở tunnel khi cần thiết. Đóng tunnel (Ctrl+C) ngay sau khi xong việc.

Test mobile app với local API

App mobile (React Native, Flutter) cần URL HTTPS cố định để gọi API backend local khi test trên thiết bị thật: 

# Backend đang chạy ở port 8080
htn-tunnel http 8080:my-api
# → https://my-api.33.id.vn

# Trong app mobile, set base URL:
# const API_URL = "https://my-api.33.id.vn"

# Jupyter Notebook — share với team:
jupyter notebook   # → localhost:8888
htn-tunnel http 8888:notebook
# → https://notebook.33.id.vn

Tự host server #

Khi nào nên tự host?

  • Enterprise / privacy — Bạn không muốn traffic đi qua server bên thứ ba.
  • Custom domain — Muốn dùng domain riêng như tunnel.company.com.
  • Team nội bộ — Setup 1 lần, cả team dùng chung server không phụ thuộc dịch vụ ngoài.
  • Hiệu năng cao — Server gần team hơn, latency thấp hơn.

Yêu cầu: VPS Linux (Ubuntu 20.04+), domain với DNS trên Cloudflare.

Cài đặt 3 bước

1

Download binary server

# Linux x64
curl -Lo htn-server \
  https://github.com/nhh0718/htn-tunnel/releases/latest/download/htn-server-linux-amd64
chmod +x htn-server
sudo mv htn-server /usr/local/bin/

# Verify
htn-server --version
2

Chạy wizard setup

Wizard tự động: cấu hình domain, lấy wildcard SSL qua Cloudflare DNS-01, cài nginx reverse proxy, tạo systemd service. Bạn chỉ cần trả lời vài câu hỏi.

sudo htn-server init

# Wizard sẽ hỏi:
# ? Domain: tunnel.yourcompany.com
# ? Cloudflare API Token: ••••••••
# ? Admin email: admin@yourcompany.com
#
# Tự động:
# ✓ DNS challenge → wildcard cert issued
# ✓ nginx configured
# ✓ systemd service created
# ✓ htn-server started
3

Chia sẻ với team

# Trên máy mỗi thành viên team:
htn-tunnel setup
# Nhập: tunnel.yourcompany.com:4443

# Dùng ngay:
htn-tunnel http 3000

Server CLI commands:

LệnhMô tả
htn-server initInteractive wizard setup lần đầu
htn-server startKhởi động server thủ công
htn-server statusXem trạng thái và danh sách tunnel đang chạy
htn-server upgradeAuto-update từ GitHub Releases
htn-server backupTạo backup users, keys, config
htn-server restoreRestore từ backup — dùng khi migrate

Vì sao cần Cloudflare DNS-01?

Tunnel dùng subdomain dạng *.tunnel.yourcompany.com (wildcard). Let's Encrypt chỉ cấp wildcard certificate qua DNS-01 challenge — nghĩa là Let's Encrypt kiểm tra bạn có quyền kiểm soát DNS record của domain không, thay vì HTTP challenge. Cloudflare API Token cho phép wizard tự tạo TXT record _acme-challenge để pass challenge này, hoàn toàn tự động.

ℹ
Cloudflare token cần quyền gì?

Token chỉ cần permission Zone → DNS → Edit cho domain của bạn. Không cần toàn bộ account permission.

Troubleshooting #

"local connection refused"

Nguyên nhân: Ứng dụng local chưa chạy hoặc đang listen ở port khác.

# Kiểm tra app local có đang chạy không:
curl http://localhost:3000

# Kiểm tra port đang listen:
netstat -tlnp | grep 3000   # Linux/macOS
netstat -ano | findstr 3000  # Windows

WebSocket không hoạt động

Nguyên nhân: Dùng phiên bản htn-tunnel cũ hơn 0.4.6.

# Update lên phiên bản mới nhất
npm update -g htn-tunnel

# Kiểm tra version
htn-tunnel --version   # Cần >= 0.4.6

"Subdomain không thuộc tài khoản"

Nguyên nhân: Bạn đã logout hoặc key hết hạn. Chạy lại htn-tunnel login để làm mới token.

Dashboard không hiện live log

Nguyên nhân: Browser cũ không hỗ trợ Server-Sent Events (SSE), hoặc bạn đang dùng ad-blocker chặn request đến server.

  • Thử browser khác (Chrome/Firefox phiên bản mới nhất)
  • Tắt ad-blocker hoặc thêm exception cho domain
  • Kiểm tra console browser (F12) xem có lỗi gì không

Xem log chi tiết

# Xem log realtime của tunnel CLI (verbose mode):
htn-tunnel http 3000 --verbose

# Xem log server (systemd) nếu self-host:
sudo journalctl -u htn-server -f

# Xem log file trực tiếp:
tail -f /var/log/htn-server/server.log

"connection refused" từ tunnel URL

Nếu URL tunnel trả về lỗi connection refused (không phải "local connection refused"):

  • Kiểm tra kết nối internet của máy bạn
  • Kiểm tra firewall/VPN không chặn port 4443
  • Thử ping 33.id.vn xem server có reachable không
  • Nếu dùng corporate network, port 4443 có thể bị block — thử mạng khác

Nâng cao #

Config file: ~/.htn-tunnel/config.yaml

Tất cả cài đặt được lưu ở file YAML này. Bạn có thể sửa thủ công nếu cần: 

# ~/.htn-tunnel/config.yaml
server: 33.id.vn:4443       # Địa chỉ server tunnel
token: htk_xxxxxxxxxxxx     # API key (sau khi login)
log_level: info              # debug | info | warn | error

Environment variables

Thay vì config file, bạn có thể dùng environment variable — hữu ích trong CI/CD hoặc Docker: 

# Override server và token qua env vars
export HTN_SERVER=33.id.vn:4443
export HTN_TOKEN=htk_xxxxxxxxxxxx

htn-tunnel http 3000

# Hoặc inline:
HTN_SERVER=33.id.vn:4443 HTN_TOKEN=htk_xxx htn-tunnel http 3000

# Dùng trong GitHub Actions:
# env:
#   HTN_TOKEN: ${{ secrets.HTN_TOKEN }}

Override qua CLI flags

# Chỉ định server khác cho lần này
htn-tunnel http 3000 --server other-server.com:4443

# Dùng token khác
htn-tunnel http 3000 --token htk_another_key

# Kết hợp
htn-tunnel http 3000:myapp --server my-vps.com:4443 --token htk_team_key

Reconnect behavior & heartbeat

htn-tunnel tự động reconnect khi mất kết nối với server. Cơ chế:

  • Heartbeat — Client gửi ping mỗi 30 giây để giữ kết nối sống. Server phát hiện client ngắt kết nối trong vòng ~60 giây.
  • Exponential backoff — Khi mất kết nối, retry sau 1s → 2s → 4s → 8s → ... tối đa 60s. Tránh spam reconnect khi server đang down.
  • Persistent subdomain — Sau khi reconnect, subdomain cũ được giữ nguyên (với tài khoản đã đăng nhập).

yamux multiplexing

htn-tunnel dùng yamux (Yet Another Multiplexer) để chạy nhiều HTTP request đồng thời trên một WebSocket connection. Lợi ích:

  • Không tốn overhead tạo kết nối TCP mới cho mỗi request
  • TLS handshake chỉ xảy ra một lần khi kết nối ban đầu
  • Hỗ trợ concurrent requests hiệu quả — phù hợp với browser hiện đại mở nhiều request song song
  • Độ trễ thấp hơn đáng kể so với HTTP/1.1 keep-alive truyền thống
